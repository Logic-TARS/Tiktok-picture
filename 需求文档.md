# 抖音二次元素材自动发布系统需求文档

## 1. 项目定位

本系统是一个本地运行的抖音二次元内容自动发布工具。

用户本地已经准备好图片、视频、封面等素材，系统不负责图片生成、图片美化、版权判断或素材来源管理。首版重点解决“选择本地素材 -> DeepSeek 生成文案 -> 可选短视频混剪 -> 自动发布到抖音”的流程。

## 2. 核心目标

1. 读取本地已有图片、视频和封面素材。
2. 使用 DeepSeek 自动生成抖音标题、正文和话题标签。
3. 支持图片发布、视频发布、视频封面设置。
4. 支持把多张图片自动混剪成短视频。
5. 通过本地浏览器自动化完成抖音发布流程。
6. 保留发布记录，方便查看成功、失败和待人工处理任务。

## 3. 已确认首版范围

已确认方案：

- 运行方式：本地工具。
- 发布入口：抖音创作者服务平台。
- 发布类型：多图图集。
- 分组规则：默认每 4 张图片生成 1 个发布任务。
- 发布时机：立即发布。
- 发布模式：半自动发布，系统自动打开发布页、上传图片、填写文案和话题，停在最终发布前由用户人工确认。
- AI 服务商：DeepSeek。
- 素材来源：不做判断，不做版权或来源管理。
- 图片处理：首版不做图片生成、美化、裁剪或二次处理。
- 视频能力：后续支持视频封面和短视频混剪，首版代码预留接口。

### 3.1 必须包含

- 本地素材选择
- 发布任务创建
- DeepSeek 文案生成
- 抖音创作者服务平台半自动发布
- 发布状态记录
- 失败原因记录
- 本地配置管理

### 3.2 不包含

- AI 生图
- 图片二次处理
- 图片版权检测
- 素材来源判断
- 图片美化、裁剪、增强
- 多平台分发
- 抖音评论、私信、粉丝管理
- 抖音数据复盘
- 云端多人协作

## 4. 目标用户

- 抖音二次元内容号运营者
- 二次元图片、壁纸、头像、图集类账号
- 需要批量发布本地素材的个人创作者
- 需要把图片素材快速转成短视频发布的运营人员

## 5. 使用流程

### 5.1 首版图片图集发布流程

1. 用户打开本地工具。
2. 选择本地图片文件或图片文件夹。
3. 填写内容主题，例如“二次元头像”“银发少女壁纸”“治愈系插画”。
4. 选择文案风格。
5. 系统调用 DeepSeek 生成标题、正文和话题标签。
6. 用户确认或修改文案。
7. 系统打开本地浏览器并进入抖音发布页面。
8. 系统按 4 张一组上传图片图集，填写文案和话题。
9. 系统停在发布前确认页，由用户人工检查并点击发布。
10. 系统记录发布结果。

### 5.2 后续视频发布流程

1. 用户选择本地视频文件。
2. 用户选择封面图片，或从视频中截取封面。
3. 系统调用 DeepSeek 生成标题、正文和话题标签。
4. 系统自动上传视频。
5. 系统自动设置封面、填写文案和话题。
6. 系统执行发布或等待用户最终确认。
7. 系统记录发布结果。

### 5.3 后续图片混剪短视频流程

1. 用户选择多张图片。
2. 用户选择混剪模板。
3. 系统生成短视频。
4. 用户选择封面。
5. 系统生成文案。
6. 系统自动发布短视频。

## 6. 功能需求

### 6.1 本地素材管理

需求：

- 支持选择单个文件。
- 支持选择文件夹。
- 支持识别图片和视频。
- 支持按文件名、创建时间或手动顺序排序。
- 支持跳过已发布素材。

支持格式：

- 图片：JPG、JPEG、PNG、WEBP
- 视频：MP4、MOV、M4V
- 封面：JPG、JPEG、PNG、WEBP

验收标准：

- 用户可以选择本地素材目录并看到待发布文件列表。
- 系统能区分图片、视频和封面。
- 已发布文件不会重复发布，除非用户手动选择重新发布。

### 6.2 DeepSeek 文案生成

DeepSeek 用于生成：

- 抖音标题
- 正文文案
- 话题标签
- 评论引导
- 可选发布建议

输入项：

- 内容主题
- 账号定位
- 素材类型：图片、图集、视频、混剪视频
- 文案风格
- 关键词
- 禁用词
- 话题数量

文案风格：

- 治愈收藏向
- 情绪共鸣向
- 头像壁纸向
- 反差钩子向
- 人设故事向
- 评论互动向

DeepSeek 配置：

- `DEEPSEEK_API_KEY`
- `DEEPSEEK_BASE_URL`
- `DEEPSEEK_MODEL`

默认模型建议：

- `deepseek-v4-flash`

验收标准：

- 配置 DeepSeek Key 后可以生成文案。
- 文案内容会根据主题和风格变化。
- 文案可以手动编辑。
- 编辑后的文案用于最终发布。

### 6.3 自动发布

首版采用本地浏览器自动化方式发布。

基本逻辑：

1. 用户提前在本地浏览器登录抖音创作者后台。
2. 系统复用本地登录态或启动独立浏览器用户数据目录。
3. 系统打开抖音发布页面。
4. 系统自动上传素材。
5. 系统自动填写标题、正文、话题、封面等字段。
6. 系统根据配置选择“立即发布”或“发布前人工确认”。
7. 系统记录发布结果。

发布模式：

- 首版固定为半自动发布：系统填写完成后停留在确认页，由用户手动点击发布。
- 后续可增加全自动发布：系统填写完成后直接点击发布。

边界说明：

- 系统不绕过验证码、风控、人脸验证或短信验证。
- 遇到登录失效、验证码、账号异常时，系统暂停并提示用户处理。
- 抖音页面结构变化可能导致自动化脚本需要更新。
- 首版不自动点击最终发布按钮，确认稳定后再开启全自动发布。

验收标准：

- 登录状态正常时，系统能自动进入发布页面。
- 系统能自动上传图片或视频。
- 系统能自动填写文案和话题。
- 半自动发布模式下，页面停在用户可检查的位置。
- 发布失败时，有明确失败原因。

### 6.4 图片发布

需求：

- 首版以多图图集发布为主。
- 默认每 4 张图片生成 1 个图集任务。
- 支持为同一组图片生成一条文案。
- 支持按文件夹扫描并自动分组。

分组规则：

- 默认模式：按文件名排序后，每 4 张图片拆分为 1 个图集任务。
- 最后一组不足 4 张时仍可生成任务，但页面需要标记数量不足。
- 已发布图片默认跳过，避免重复发布。

验收标准：

- 用户可以将一个文件夹中的图片按组发布。
- 系统能记录每组图片对应的文案和发布状态。

### 6.5 视频发布与封面

需求：

- 支持上传本地视频。
- 支持选择本地图片作为视频封面。
- 支持从视频指定时间点截取封面。
- 支持封面预览。
- 支持封面与视频一起进入发布流程。

验收标准：

- 用户可以为每个视频指定封面。
- 未指定封面时，系统可以使用默认截帧或跳过封面设置。
- 发布记录中能看到视频文件、封面文件和文案。

### 6.6 图片混剪短视频

需求：

- 支持多张图片生成短视频。
- 支持设置每张图片展示时长。
- 支持设置总视频时长。
- 支持添加转场效果。
- 支持添加背景音乐。
- 支持选择输出分辨率。
- 支持生成视频封面。

首版混剪参数：

- 输出比例：9:16
- 默认分辨率：1080 x 1920
- 每张图片默认时长：2 秒
- 默认转场：淡入淡出
- 默认输出格式：MP4

可选模板：

- 壁纸图集
- 头像合集
- 治愈慢切
- 快节奏卡点
- 赛博霓虹

技术建议：

- 使用 FFmpeg 进行视频生成。
- 本地检测 FFmpeg 是否可用。
- 未安装 FFmpeg 时给出安装提示。

验收标准：

- 多张图片可以生成一个 MP4 短视频。
- 生成视频可以被发布流程识别。
- 视频封面可以使用第一张图片或用户指定图片。

### 6.7 任务队列

需求：

- 支持创建多个待发布任务。
- 支持暂停、继续、取消。
- 支持失败重试。
- 支持跳过已发布素材。
- 支持发布间隔设置，降低连续操作风险。

任务状态：

- 待处理
- 文案生成中
- 待确认
- 发布中
- 发布成功
- 发布失败
- 需要人工处理
- 已跳过

验收标准：

- 用户可以看到每个任务当前状态。
- 失败任务可以单独重试。
- 系统关闭后，历史任务仍可查看。

### 6.8 发布记录

每条发布记录保存：

- 任务 ID
- 素材路径
- 素材类型
- 封面路径
- 标题
- 正文
- 话题标签
- 发布时间
- 发布模式
- 发布状态
- 失败原因
- 抖音返回页面或作品链接，若能获取

验收标准：

- 用户可以查看历史发布记录。
- 用户可以搜索文件名或状态。
- 记录可导出为 CSV 或 JSON。

## 7. 页面需求

首版建议做一个本地 Web 工作台。

页面模块：

- 配置区：DeepSeek Key、模型、发布模式、浏览器配置。
- 素材区：选择文件、选择文件夹、素材列表。
- 文案区：主题、风格、关键词、生成结果、手动编辑。
- 混剪区：图片时长、模板、音乐、输出设置。
- 任务区：任务队列、状态、重试、暂停。
- 记录区：历史发布记录。

核心按钮：

- 选择素材
- 生成文案
- 生成混剪视频
- 创建发布任务
- 开始发布
- 暂停发布
- 重试失败

- 支持测试浏览器启动。

## 9. 技术方案建议

建议技术栈：

- 后端：Python + FastAPI
- 前端：原生 HTML/CSS/JavaScript
- 浏览器自动化：Playwright
- AI 文案：DeepSeek API
- 视频混剪：FFmpeg
- 本地数据：SQLite
- 配置文件：`.env` 或本地 JSON 配置

选择理由：

- 本地工具部署简单。
- Python 适合串联 AI、浏览器自动化和 FFmpeg。
- Playwright 适合自动填写网页发布流程。
- SQLite 足够保存本地任务和发布记录。
- FFmpeg 适合稳定生成短视频。

## 11. 接口需求

建议接口：

- `GET /`：打开工作台。
- `GET /api/health`：健康检查。
- `GET /api/config`：读取配置。
- `POST /api/config`：保存配置。
- `POST /api/deepseek/test`：测试 DeepSeek 连接。
- `POST /api/materials/scan`：扫描本地素材。
- `POST /api/captions/generate`：生成文案。
- `POST /api/videos/mix`：图片混剪成视频。
- `POST /api/jobs`：创建发布任务。
- `GET /api/jobs`：获取任务列表。
- `POST /api/jobs/{job_id}/start`：开始发布任务。
- `POST /api/jobs/{job_id}/retry`：重试任务。
- `POST /api/jobs/pause`：暂停发布队列。
- `GET /api/records`：获取发布记录。
- `GET /api/records/export`：导出发布记录。

## 13. 自动发布风险与处理策略

可能风险：

- 抖音登录状态过期。
- 页面结构变化。
- 上传控件变化。
- 账号触发风控。
- 发布频率过高。
- 网络异常。

处理策略：

- 首版默认半自动发布。
- 每次发布之间增加间隔。
- 失败后保留现场浏览器页面。
- 记录具体失败步骤。
- 用户处理验证码或登录后可以继续任务。
- 自动化选择器集中维护，便于后续修复。
